# Bird Eats Bug

## Kategoria

- Główna kategoria: `bug-reporting`
- Podkategoria: `screen-recording`
- Darmowe: `True`
- Poziom trudności: `easy`
- Wartość dla manualnego testera: `high`
- Wartość dla automatyzacji: `low`
- Wymaga kodowania: `False`

## Do czego służy

Bird Eats Bug nagrywa ekran i automatycznie zbiera logi przeglądarki oraz requesty sieciowe w trakcie nagrania. Idealne do dokumentowania bugów w webapplikacjach, gdy "cannot reproduce" wynika z braku danych technicznych.

## Najlepsze zastosowania

- Bug report dla webappów z automatycznym network trace bez otwierania DevTools
- Nagranie kroków reprodukcji z logami konsoli w jednym pliku
- Dokumentowanie błędów wizualnych i funkcjonalnych w aplikacjach webowych
- Szybkie tworzenie raportów bez potrzeby nagrywania i kopiowania logów osobno

## Kiedy używać

Używaj tego narzędzia wtedy, gdy testujesz aplikację webową i chcesz do bug raportu dołączyć nie tylko wideo z krokami, ale też automatycznie zebrane logi i ruch sieciowy — bez ręcznego otwierania zakładki Network w DevTools.

## Pierwszy praktyczny workflow

1. Zainstaluj rozszerzenie Bird Eats Bug ze strony birdeatsbug.com lub Chrome Web Store.
2. Przejdź do strony/aplikacji z bugiem i kliknij ikonę rozszerzenia.
3. Uruchom nagranie przyciskiem "Start recording".
4. Odtwórz kroki prowadzące do buga — Bird Eats Bug zbiera logi i sieć w tle.
5. Zatrzymaj nagranie i skopiuj link do raportu lub pobierz go do załączenia w Jirze.

## Następny krok

Porównaj raporty Bird Eats Bug z Jam — sprawdź, które narzędzie lepiej integruje się z waszym trackerem błędów i generuje bardziej czytelny network log.

## Ryzyka i ograniczenia

- Nie traktuj narzędzia jako celu samego w sobie.
- Sprawdź licencję przed użyciem komercyjnym.
- Narzędzie działa wyłącznie w przeglądarkach — nie obsługuje natywnych aplikacji mobilnych.
- Nagrania mogą zawierać dane wrażliwe użytkowników — zawsze anonimizuj przed udostępnieniem.
