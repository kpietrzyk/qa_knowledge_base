# Bug Reporting & Adnotacje

Narzędzia do szybkiego, kompletnego dokumentowania błędów z dowodami technicznymi.

## Dlaczego to ważne

Dobry bug report = szybka naprawa. Złe dowody = "cannot reproduce".

Cel: 1 klik lub kilka komend → nagranie + logi + kontekst sieciowy + ticket w Jira.

## Narzędzia w tej kategorii

- **Jam** — najszybszy sposób: 1 klik = nagranie + logi + network + Jira
- **Bird Eats Bug** — jak Jam, skupiony na webapps
- **Greenshot** — screenshoty z adnotacjami (strzałki, obramowania, tekst)
- **Loom** — nagranie wideo z komentarzem głosowym do Jiry
- **Instabug** — SDK w buildzie testowym: potrząśnięcie = raport z pełnym kontekstem

## Kiedy którego użyć

| Sytuacja | Narzędzie |
|---|---|
| Bug w aplikacji mobilnej, potrzebujesz logów automatycznie | Jam |
| Bug w webapp, chcesz network trace | Bird Eats Bug |
| Szybka adnotacja screenshota | Greenshot |
| Długi scenariusz z komentarzem głosowym | Loom |
| Build testowy, testerzy w terenie | Instabug SDK |
