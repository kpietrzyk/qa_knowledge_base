# QA daily workflow

## Start dnia

- [ ] Sprawdź Jira / board
- [ ] Sprawdź priorytety
- [ ] Sprawdź nowe buildy
- [ ] Sprawdź znane blokery

## Podczas testów

- [ ] Zbieraj dowody od razu
- [ ] Zapisuj dane testowe
- [ ] Notuj wersję aplikacji i środowisko
- [ ] Przy błędzie API sprawdź request/response
- [ ] Przy crashu sprawdź logcat/Crashlytics/Sentry

## Koniec dnia

- [ ] Zaktualizuj bugi
- [ ] Zaktualizuj test run
- [ ] Wypisz ryzyka
- [ ] Przygotuj pytania do PO/devów
