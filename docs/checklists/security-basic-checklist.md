# Checklist — security basics dla QA mobile

To nie zastępuje pentestu. To minimalna checklista QA.

- [ ] Brak sekretów w widocznych logach
- [ ] Tokeny nie są pokazywane w UI
- [ ] Sesja wygasa zgodnie z wymaganiami
- [ ] Endpointy wymagają autoryzacji
- [ ] Użytkownik A nie widzi danych użytkownika B
- [ ] Błędy API nie ujawniają stack trace
- [ ] Dane wrażliwe nie są niepotrzebnie zapisywane lokalnie
- [ ] Root/jailbreak behavior jest znany
- [ ] Certyfikaty i SSL są poprawnie obsługiwane
