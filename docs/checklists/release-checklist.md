# Checklist — release aplikacji mobilnej

## Przed releasem

- [ ] Zakres zmian znany
- [ ] Build oznaczony wersją
- [ ] Release notes przygotowane
- [ ] Lista ryzyk przygotowana
- [ ] Smoke test wykonany
- [ ] Krytyczne bugi zamknięte albo zaakceptowane

## Testy

- [ ] Instalacja czysta
- [ ] Update z poprzedniej wersji
- [ ] Logowanie
- [ ] Główne flow biznesowe
- [ ] Płatności / zamówienia / krytyczne akcje
- [ ] Offline/online
- [ ] Push notifications
- [ ] Crashlytics/Sentry sprawdzone

## Po releasie

- [ ] Monitoring crashy
- [ ] Monitoring błędów backendu
- [ ] Feedback użytkowników
- [ ] Hotfix plan
