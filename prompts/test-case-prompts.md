# Prompty do przypadków testowych

## Generowanie przypadków z wymagania

```text
Jesteś QA Engineerem. Na podstawie wymagania wygeneruj przypadki testowe.

Podziel na:
- happy path,
- negative cases,
- edge cases,
- permission cases,
- network cases,
- mobile-specific cases,
- API/backend cases,
- accessibility basics,
- regression risks.

Wymaganie:
[WKLEJ]
```

## Redukcja przypadków do smoke testów

```text
Z tej listy przypadków testowych wybierz minimalny smoke suite.
Załóż, że mamy 20 minut na testy przed releasem.
Podaj:
- testy krytyczne,
- testy opcjonalne,
- uzasadnienie.
Lista:
[WKLEJ]
```

## Zamiana manual test case na automatyzację

```text
Przeanalizuj ten manualny przypadek testowy i oceń, czy nadaje się do automatyzacji.

Oceń:
1. stabilność,
2. wartość regresyjną,
3. trudność automatyzacji,
4. wymagane dane testowe,
5. rekomendowane narzędzie: Maestro, Appium, Playwright czy API test.

Test case:
[WKLEJ]
```
